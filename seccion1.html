En qué consiste una Página Web

Las páginas Web son para fines prácticos documentos HTML que se despliegan al usuario en un navegador web, dependiendo el sitio que estemos visitando puede haber un sistema detrás generando el contenido (wordpress, Drupal, Blackboard, etc), o pueden ser simplemente archivos html, no importa lo que haya detrás (backend) el usuario final sólo verá archivos html en su navegador, sin importar si fueron creados dinámicamente o no, para el navegador todo esto es igual.

Dado que solo nos interesa la parte que se despliega al usuario final, es decir el documento HTML al que apunta su navegador, este documento hará referencia a diferentes archivos y contendrá diferentes partes de contenido para desplegar la página. Ahora podemos dividir los archivos que componen una página web en 3 partes:

  1. Contenido, esto es el HTML donde estará el texto, y la referencia a las imágenes, audio, videos, etc que componen la información que contiene la página, estos últimos no están dentro del HTML sino que este solo apunta hacia ellos y los despliega.
  2. Presentación o CSS, las CSS (Cascade Style Sheets) o Cascadas de Estilo, son encargadas de la presentación de la página, aquí están los estilos de texto, los colores, fondos, tipos de letra y demás caracteristicas visuales del documento, estas también se usan para definir la maquetación o layout de la página; no se preocupen hablaremos mas acerca de estas en un futuro.
  3. Interacción, este es usualmente un archivo de JavaScript donde estan las instrucciones para las diferentes interacción que el usuario tendrá con nuestro sitio así como algunas animaciones

Como verán una página web está compuesta por diferentes elementos y archivos, acerca de los cuales hablaremos en esta guía.

Definción de HTML

HTML Hyper Text Markup Language (Lenguaje de Marcación de Documentos para Hipertexto) es básicamente un conjunto de etiquetas que se utiliza para establecer el contenido y la estructura de las páginas web.

Este Lenguaje de Marcación se define como tal porque utiliza etiquetas (tags en Inglés) para encapsular el contenido y describir diferentes características de este, un ejemplo de una etiqueta sería:

{% highlight html %} Negritas {% endhighlight %}

Que indica que el texto de la etiqueta se presente en negritas, no se preocupen por entender la estructura de la etiqueta por el momento, más adelante profundizaremos en ese concepto.

En lo que respecta los archivos HTML son simplemente archivos de texto con la particularidad de que usan las extensiones .htm y .html, por eso solo necesitamos un editor de texto para escribir y crear una página web. Es recomendable usar editores sencillos y ligeros como Atom y Brackets que son un par de herramientas gratuitas y bastante útiles para desarrollar html.

Anatomia de las etiquetas HTML

Los elementos de html están usualmente delimitados por dos etiquetas, una de apertura y otra de cierre, que deben escribrise siempre en minúsculas, deb siendo la principal diferencia la / (slash) en la etiqueta de cierre. Estas etiquetas encapsulan el contenido y definen el comportamiento que este tendra.

Atributos de un elemento

Una característica importante de las etiquetas de HTML, son los atributos, que se localizan en la etiqueta de apertura, y para proveen información adicional respecto al elemento que los contiene, esta información puede ser para modificar el comportamiento, el estilo o la interactividad bajo ciertas circunstancias. Los atributos están compuestos de dos partes, nombre y valor, ambos es recomendable escribirlos siempre en minúsculas al igual que las etiquetas.

En ejemplo anterior el atributo “id” sirve para darle un nombre unico que identifique al elemento.

La mayoría de los atributos están restringidos a ciertos elementos, aunque muchos de ellos pueden utilizarse en todos los elementos; también es importante mencionar que ciertos valores tienen un formato predefinido que exploraremos conforme avancemos en esta guía.

Estructura de un Documento HTML

Un documento HTML requiere de cierta estructura para que pueda ser interpretado correctamente por los diferentes navegadores, esta a su vez esta compuesta por varias partes :

HTML:Esta etiqueta encapsula las demás etiquetas y le dice al navegador que el documento es un HTML
Head delimitado por la etiqueta <head>: Cabeza o cabecera, aqui se colocan diversas etiquetas informativas, respecto al documento, como el tíitulo, el tipod de documento, referencias a otros archivos, etc. El contenido de esta no aparece en le navegador y no es visto por el usuario, pero sirve a los navegadores y otros sistemas para encontrar información útil para entender y procesar el documento.
Body delimitado por la etiqueta <body>: Esto es lo que ve el usuario, aqui esta todo el contenido, texto, imágenes, sonido, video, etc

Creando Nuestra primer Pagina HTML

Editores de HTML

Antes de crear nuestro primer documento de HTML primero debemos seleccionar un editor para hacerlo; existen dos tipos de editores de HTML los editores What You See Is What Get,WYSIWG, que son editores con una interfaz gráfica donde simplemente nos limitamos a arrastrar los diferentes elementos que necesitamos para construir nuestro documento y el programa se encarga de transformar esto a código HTMl, ahora el problema con este tipo de software es que crea mucho código innecesario y que al momento de realizar cambios es muy díficil interpretar lo que el software ha creado esto complica mucho el mantenimiento, además de que este tipo de programas limita muhco nuestras opciones.

Otro tipo de editores de HTML son los editores de texto especializados, estos son mucho más simples en cuanto a sus elementos pero requieren un mayor conocimiento de HTML, ya que tenemos que escribir todo prácticamente desde cero, pero nos da la ventaja de que sólo incluiremos lo que necesitamos y que tenemos total control sobre el documento generado.

Dado que que esto es una guia de HTML optaremos por el editor de texto, ahora hay muchas opciones gratuitas y pagadas de estos editores pero para esta guía usaremos ATOM , ya que es ligero, gratis, tiene opciones de personalización muy interesantes y funciona tanto en MAC como en PC.

Juego de Caracteres

El juego de caracteres le indica al navegador la colección de símbolos que requiere para desplegar la página correctamente, me refiero a los caracteres como letras acentuadas, diéresis, y símbolos de diversos idiomas; cuando este juego de caracteres no esta especificado podemos encontrarnos con problemas como que algunas letras no se despliegan o no podemos ver acentos y signos de puntuación particulares de nuestro idioma.

Este juego de caracteres, o codificación, depende del sistema operativo que estás usando para crear tu archivo HTML. Mientras que unos sistemas como Linux o Mac usan por defecto un juego de caracteres llamado UTF-8, en Windows se usa de manera predeterminada otro juego de caracteres llamado ISO-8859-1. Es otro de los motivos por los que en pasados artículos recomendábamos Brackets o Atom como editores de código, ya que éstos trabajan siempre en UTF-8, independientemente del sistema operativo. 

El juego predeterminado de HTML 5 es UTF-8 y cubre la gran parte de los símbolos a nivel mundial. Este juego de caracteres se especifica dentro de una etiqueta meta la cual abordaremos más adelante pero por ahora basta con decir que este tipo de etiquetas especifican características propias del documento.
